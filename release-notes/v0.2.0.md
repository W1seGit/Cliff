# Cliff v0.2.0

This release focuses on making imported Minecraft servers feel reliable from the first run.

## Importing servers

- Added a clearer import progress UI with staged detection and import status.
- Imported servers now default to auto-managed Java instead of a PATH Java command.
- Imported servers now default to `nogui`, so Minecraft's desktop server window does not open during launch.
- Direct jar launches avoid adding a duplicate `nogui` argument when it is already configured.

## Startup behavior

- Imported jar launches now stay headless by default, which avoids the Minecraft server GUI keeping a failed launch visible as active until that window is closed.

## Releases

- GitHub releases now use a written release message for this version instead of only auto-generated commit notes.
